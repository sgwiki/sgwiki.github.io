## 1. 섹션 6 — 소스별 공개 표시 방침 vs 저작권 분석 일관성

### 1-1. [CRITICAL] `sg_game_sge`가 공개 방침표에 등재됨 — 배제 결정과 정면 모순
- 섹션 3 dataforge filter 표: "`sg_game_sge`(한글 패치): 저작권 리스크로 **배제**".
- 저작권 doc §1: `sg_game_sge` = "Very High risk → **배제 권장**, 절대 금지".
- 그러나 섹션 6 표은 `sg_game_sge`를 "`⚠️ 검열 / 슈타인즈 게이트 엘리트`"로 등재.
- **문제**: RAG에서 배제된 소스는 공개 방침이 존재할 수 없음. 표에서 제거하거나, "사용 금지(배제)" 행으로 명시해야 함. "⚠️ 검열" 표기는 여전히 사용 중인 것으로 오해 유발.
- **추가**: 표시명 `슈타인즈 게이트 엘리트`도 사실 관계 불일치 의심. 섹션 3·저작권 doc은 `sge`를 원작 슈타인즈게이트의 "한글 패치"로 정의(ELITE는 별개 작품). 네이밍 재확인 필요.

### 1-2. [CRITICAL] `namuwiki` 출처 미표시(❌) → CC BY-NC-SA 2.0 KR BY 의무 위반 가능
- 저작권 doc §4: 나무위키 BY(Attribution) = "출처 URL + 라이선스 표기 **필수**", "`sources/` YAML `url`+`last_modified`로 **이미 충족 ✅**".
- 그러나 섹션 6 표은 namuwiki를 "❌ 산문 처리, **출처 미표시**"로 분류 → 설계 의도와 정면 충돌.
- AI가 **완전 재작성**하면 저작권법상 2차 저작물이 아니라 BY가 면제된다는 해석은 가능하나, (a) 그 판단 기준이 문서에 없고, (b) 저작권 doc은 명시적으로 출처 표시를 전제로 "✅ 충족"이라 함.
- **권고**: namuwiki 행을 "재작성 필수 + 출처 URL 표시(BY 준수)"로 변경하거나, 재작성으로 BY 면제를 주장할 근거를 명시. 위키 자체를 CC BY-NC-SA 2.0 KR로 명시(저작권 doc 권고)하는 것도 보완책.

### 1-3. [WARNING] `sg_game_sg0_en` "검열" 표시의 의미 불명확 — "원문 인용 금지" 누락
- 저작권 doc §3 핵심 제약: sg_game_sg0_en은 "**원문 절대 미노출**, 출처 표시".
- 섹션 6 표은 "`⚠️ 검열 / 슈타인즈 게이트 제로`"로만 표기.
- "검열"이 (a) 소스명을 라벨로만 표시, (b) 원문 chunk를 재작성/요약해야 함 중 어느 쪽인지 불명확. 섹션 6 인용 예시에는 스크립트 원문을 인용블록에 넣어도 되는지에 대한 규정이 없음.
- 저작권 doc 1순위 원칙("원문 청크 미표시")이 섹션 6 정책에 명시적으로 반영되지 않음.

### 1-4. [WARNING] `sg_paper`(팬 논문)이 저작권 검토 대상에서 누락
- 섹션 6 표은 sg_paper를 "✅ 논문 제목 그대로" 공개 허용.
- 그러나 저작권 doc은 sg_paper를 **별도 분석하지 않음**(팬 자막·한글 패치만 분석).
- 팬 논문도 저작권자가 별도인 2차 저작물. 섹션 6 인용 예시 `[팬 분석] "..."`처럼 실제 구문을 인용하면 팬 저작권자의 표현 침해 소지.
- ✅ 전면 공개보다는 "짧은 인용 + 출처(제목·저자·버전)" 제한으로 명시 권장. 저작권 doc에 sg_paper 분석 추가 필요.

### 1-5. [WARNING] `qaset_with_rag`의 분류 근거 미정리
- 섹션 6은 qaset을 ❌(산문 처리, 출처 미표시)로 분류 — 보수적이라 안전.
- 그러나 qaset이 공식 QA자료집 기반임에도 `reference/official/`(✅)과 다르게 취급되는 근거가 문서화되지 않았고, 저작권 doc에도 qaset 별도 분석이 없음. 보수적 결정의 사유를 한 줄 메모로 보강 권장.

### 1-6. [OK] `reference/official/`(공식 자료집) ✅
- 저작권 doc §5(Low risk, 짧은 인용 + 출처)과 일치.

### 1-7. [OK] `sg-ontology` ❌
- 팀 자체 SPARQL 온톨로지 산물이므로 출처 미표시 무방. 단, 저작권 doc에 명시 분석은 없으니 보강 권장.

### 1-8. [WARNING] 인용 예시 태그가 `[공식]`/`[팬 분석]` 2종만 — 소스 유형별 매핑 부재
- 인용 블록에만 태그를 단다는 원칙과, 각 소스의 공개 방침(❌/⚠️/✅) 사이의 매핑이 없음. sg_game_sg0_en·namuwiki·qaset에서 인용이 가능한지, 가능하다면 어떤 태그/포맷인지 규정 누락.

## 2. 섹션 6 — source-sanitizer 체크 항목 완전성

현재 항목: (1) chunk ID 패턴, (2) source_filter 이름, (3) 내부 파일 경로, (4) 위반 시 보고·커밋 보류.

### 2-1. [CRITICAL] "원문 청크 미노출 / 과다 인용" 검사 부재
- 저작권 doc 공통 원칙 1순위가 "원문 청크 미표시"임에도, sanitizer는 식별자(ID/경로/filter명)만 검사.
- sg_game_sg0_en·공식 QA·팬 논문의 긴 verbatim 인용, namuwiki 직접 복사를 잡지 못함.
- 권고: (a) 인용 블록 길이/비율 임계값 검사, (b) 소스 원문과 N-gram 유사도 검사 등 "과다 인용 탐지" 항목 추가.

### 2-2. [WARNING] 나무위키 BY 충족(출처 URL 표시) 검사 부재
- 1-2와 연동. namuwiki 기반 문단의 출처 URL 표시 여부를 검증하는 항목 필요.

### 2-3. [WARNING] "인용 블록 출처 필수" 검사 부재
- 섹션 4 팀장 승인 기준 "공식/팬 논문 인용 블록에 출처 있음"이 sanitizer 체크리스트에 없음. sanitizer 단에서 선검토 가능.

### 2-4. [WARNING] "스포일러 배지 존재" 검사 부재
- 섹션 4 승인 기준 "문서 상단 스포일러 배지 있음"이 sanitizer 단계에서 검증되지 않음(명시 항목에서 누락). 자동화 가능한 정형 검사이므로 sanitizer에 편입 권장.

### 2-5. [WARNING] 배제 소스(sge·팬 자막) 콘텐츠 유출 검사 부재
- chunk ID 접두사(`sge_`)는 검사하나, ID 없이 내용만 스며든 경우는 미검증. 배제 소스 원문과의 유사도 검사가 방어막이 됨(2-1과 중복 구현 가능).

### 2-6. [WARNING] 위반 로깅/감사내역 명세 부재
- "팀장에게 보고, 커밋 보류"만 있고, 위반 내역을 어디에 어떤 형식으로 저장할지 규정 없음. `suggestions/decisions/`와 별도 로그 스펙 권장.

## 3. 섹션 7 — 남은 작업 백로그 갭

사전 확인 결과: `wiki/`(lore 포함) 존재, 단 `mkdocs.yml`·`sources/`·`.github/workflows/`·`worker/`는 **미존재**. `prompts/wiki_writing_system.md`·`docker/holyclaude/{Dockerfile,docker-compose.yaml}`은 **존재**.

### 3-1. [CRITICAL] `sources/` YAML 스펙/구현 항목 누락
- 저작권 doc 전체가 `sources/` YAML(`url`, `last_modified`, `source_filter`, `note`, `claims`)에 의존하며, 심지어 "이미 설계됨 ✅"라고 단언.
- 그러나 (a) 실제 `sources/` 디렉토리 미존재, (b) 섹션 7 백로그에도 항목 없음. → 저작권 doc의 "✅ 충족" 전제가 허구.
- writer·sanitizer·judge가 읽고 갱신해야 하므로 섹션 7에 `sources/` 스키마 + 시딩 항목 필수 추가.

### 3-2. [CRITICAL] MkDocs Material 정적 사이트 구축 항목 누락
- 섹션 2 핵심 인프라: "wiki/ 정적 사이트(MkDocs Material) + 한국어 검색 + Mermaid + 스포일러 배지 + 제안 히스토리 탭".
- 그러나 `mkdocs.yml`이 없고 섹션 7에도 항목 없음. 배포 파이프라인(섹션 5 ⑦)의 전제가 무너짐.
- 항목 추가 권장: `mkdocs.yml` + 플러그인(i18n 검색, mermaid, 스포일러 배지 커스텀, 히스토리 탭).

### 3-3. [WARNING] GitHub Actions 배포 워크플로(`.github/workflows/deploy.yml`) 누락
- 섹션 5 ⑦에서 참조(`→ mkdocs build → Cloudflare Pages 배포`)하나 백로그에 없고 실제 디렉토리도 미존재.

### 3-4. [WARNING] 위키 푸터 면책 문구 + 위키 라이선스(CC BY-NC-SA 2.0 KR) 명시 누락
- 저작권 doc 권고("비상업 팬 프로젝트 면책 문구", "위키 CC BY-NC-SA 명시")가 섹션 7에 반영 안 됨.

### 3-5. [WARNING] RAG 저작권 정책 모니터링 프로세스 누락
- 저작권 doc이 sg_game_sg0_en 사용 조건으로 "정책 변화 모니터링(RAG 소송 동향)"을 요구하나 백로그에 없음.

### 3-6. [WARNING] git commit 권한 분리(팀장 독점) 구현·검증 항목 누락
- 섹션 4 핵심 통제("하위 에이전트는 파일 write만, commit은 팀장 독점")이나, 이 권한 경계를 어떻게 강제할지(컨테이너 권한, pre-commit hook 등) 백로그에 없음.

### 3-7. [WARNING] 제안 히스토리 탭 렌더링 파이프라인 구분 누락
- 섹션 5 ⑤ "히스토리 탭 반영(Pages 재배포)"의 구현이 R2 폴링(#8)과 분리되어 있지 않음. `decisions/{id}.json` → 히스토리 탭 렌더링 별도 항목 필요.

### 3-8. [WARNING] 항목 #9(기획서 양식)와 #4(CLAUDE.md)의 위치 중복
