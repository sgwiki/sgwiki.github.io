# RAG 소스 저작권 검토

> 작성일: 2026-06-19  
> 대상: holyclaude Agent 팀이 사용하는 데이터 소스의 저작권/라이선스 적법성  
> 전제: **청크 원문은 사용자에게 미표시, 출처 메타데이터(링크·제목)만 UI 노출** ← 이 결정이 전체 리스크를 대폭 낮춤

## 소스별 상세 분석

### 1. 비공식 유저 한글 패치 게임 스크립트 (`sg_game_sge`)

**리스크: Very High → 배제 권장**

- 원 저작권자: MAGES./5pb. (게임 원문)
- 팬 번역 패치 자체가 **무허가 2차 저작물** — 법적으로 원저작권 침해
- 이를 RAG 소스로 사용하면 삼중 침해 리스크:
  1. MAGES./5pb.에 대한 원저작물 직접 침해
  2. 팬 패치 제작자의 번역 저작권 침해
  3. 한국 대법원 인정 "2차 침해(방조)" 책임
- 참고: 슈타인즈 게이트 영어 팬 번역팀(Blick Winkel)은 JAST USA 공식 라이선스 추진 시 자진 철회 (Kotaku, 2014)

**대안:** 공식 영어 스크립트(`sg_game_sg0_en`) 또는 공식 자료집 사용

---

### 2. 애니메이션 팬 자막 파일

**리스크: High → 배제 권장**

- 원 저작권자: MAGES./제작위원회/방송사
- 팬 자막(팬서브)은 무허가 2차 저작물 — 국제적으로 공정이용 보호 안 받음 (ANN 법률 분석, 네덜란드 법원 판례)
- 저작권자가 두 겹(원작 제작위원회 + 팬 자막 제작자)으로 존재
- 원문 미표시라도 벡터 DB 저장 자체가 "복제"에 해당할 수 있음

**대안:** 공식 한국어/영어 자막이 있다면 그것을 사용

---

### 3. 공식 영어 게임 스크립트 (`sg_game_sg0_en`)

**리스크: High(원문 표시 시) / Medium(내부 RAG 한정 시)**

- 원 저작권자: MAGES./5pb. + Spike Chunsoft USA
- 명백히 저작권 보호 대상인 상업 출시 텍스트

**원문 UI 표시 시 (❌ 금지):**  
한국 저작권법 제28조 인용 요건(대법원 97도2227):  
"주종 관계 명확 + 수요 대체 없음 + 변형성 있음" → 위키 문단과 동등한 길이의 원문 청크를 footnote로 표시하면 "인용"이 아닌 "복제·전시"로 판단될 위험 높음

**내부 RAG 한정 시 (⚠️ 조건부 허용):**  
- 사용자에게 원문 미노출 전제
- Fair Use 4요소: 비영리/변형적 목적 ✅, 창작물 ❌, 사용량 적음 ✅, 시장 대체 없음 ✅
- 비영리 팬 위키를 Spike Chunsoft가 단속한 선례 없음
- 단, 2024~2025년 RAG 저작권 소송(Perplexity AI, Cohere 등) 진행 중 — 법리 확립 추이 모니터링 필요

**사용 조건:** 원문 절대 미노출, 출처 표시, 정책 변화 모니터링

---

### 4. 나무위키 (CC BY-NC-SA 2.0 KR)

**리스크: Medium → 조건 준수 시 사용 가능**

라이선스 의무 3가지:

| 조건 | 의무 | 이 위키 |
|------|------|---------|
| **BY** (Attribution) | 출처 URL + 라이선스 표기 필수 | sources/ YAML `url` + `last_modified` 이미 설계됨 ✅ |
| **NC** (NonCommercial) | 비영리만 허용 | 비영리 팬 위키 ✅ (광고·유료 기능 추가 시 즉시 위반) |
| **SA** (ShareAlike) | 나무위키 원문 복제 시 동일 라이선스 전이 | AI가 **참조하여 재작성**하면 SA 전이 없음 |

**핵심 원칙:**  
나무위키 텍스트를 **직접 복사하지 말 것** — AI가 내용을 참조해 새로 작성하면 아이디어는 저작권 보호 안 됨, SA 의무 없음

**위키 라이선스 권고:**  
위키 자체를 **CC BY-NC-SA 2.0 KR**로 명시 권장 (법적 의무 아님, 나무위키와 호환 + 커뮤니티 신뢰 향상)

---

### 5. 공식 자료집 (`reference/official/`)

**리스크: Low → 인용 한도 내 사용 가능**

- 공식 QA자료집, WePlay 개발자 토크, 논문 등
- 공식 출처이므로 신뢰도 최고
- 짧은 인용 + 출처 표시는 저작권법 제28조 인용으로 보호 가능
- **주의:** 전문 대량 복제는 금지. 핵심 구절 인용 + 출처 링크 형태 유지

## 청크 UI — 안전한 구현 방식

**표시할 것 (출처 메타데이터):**
```
이 문단의 근거:
  [슈타인즈 게이트 제로 공식 영어 스크립트]
  [나무위키: 오카베 린타로 (2026-03-12) ↗]
  [공식 QA자료집 §리딩 슈타이너]
```

**표시하지 않을 것:**
```
❌ "실제 원문 청크 텍스트..."
```

이 방식은 학술 논문의 주석과 동일 — 법적으로 가장 방어 가능한 형태.

## sources/ YAML 스펙 반영

```yaml
# 나무위키 — BY 의무 충족을 위해 url + last_modified 필수
- id: src-005
  type: namuwiki
  url: "https://namu.wiki/w/오카베린타로"
  last_modified: 2026-03-12        # ← 필수
  retrieved_at: 2026-06-19
  claims:
    - "..."

# 공식 영어 스크립트 — 내부 RAG 전용, 원문 미노출
- id: src-002
  type: script
  mcp: dataforge
  source_filter: sg_game_sg0_en
  note: "공식 영어판 — 원문 UI 노출 금지"
  retrieved_at: 2026-06-19
  claims:
    - "..."

# 비공식 한글 패치 — 사용 배제 권장
# sg_game_sge: 법적 리스크로 인해 사용하지 않는 것이 최선
```

## 공통 원칙

1. **원문 청크 미표시** — 모든 소스에서 이 원칙이 가장 중요한 리스크 완화책
2. **출처 메타데이터는 항상 표시** — 신뢰도 + 법적 방어
3. **AI 재작성** — 나무위키 등에서 직접 복사 금지, 항상 참조 후 재작성
4. **비영리 유지** — 광고·유료 기능 추가 시 NC 위반
5. **면책 문구** — 위키 푸터에 "비상업 팬 프로젝트, 모든 원작 저작권은 MAGES./5pb./해당 저작권자에게 귀속됨" 삽입

## 불확실한 영역

- RAG footnote 표시를 한국 법원이 저작권법 제28조 인용으로 볼지 **선례 없음** (2025년 기준)
- 나무위키 청크를 AI가 참조하여 새로 작성한 문단의 SA 전이 여부
- MAGES./Spike Chunsoft의 비영리 팬 위키 실제 집행 의지 (법적 침해 가능성 ≠ 소송 제기)

> **참고:** 이 문서는 법률 조언이 아닌 실용적 리스크 평가입니다. 완전한 법적 확실성이 필요하면 저작권 전문 변호사 상담을 권장합니다.
